Introdução: Crie e Publique Páginas no GitHub com Facilidade!

JaoGit é uma ferramenta prática para criar e publicar sites no GitHub Pages de forma rápida e simplificada. Com modelos prontos e suporte a HTML, CSS e JavaScript, ela facilita o desenvolvimento e aprimora a apresentação das suas páginas no GitHub.

Comece com o código padrão do HTML5 no arquivo index.html. No VS kbd, digite ! ou html:5 e pressione Enter para gerar a estrutura básica automaticamente. Ou copia a estrutura abaixo.

Estrutura: Descubra o que está incluído no JaoGit.

As estruturas auxiliam na organização e na busca por arquivos, cada uma com seu propósito específico, como assets e base. Cada diretório contém elementos essenciais, e futuramente novas implementações facilitarão a criação de projetos e o upload no GitHub Pages.

Assets: Área de CSS, scripts e outros recursos globais

A pasta assets contém diretórios fixos, como css/styles.css e js/scripts.js. No entanto, é possível adicionar outros arquivos globais, como imagens (img/banner.png), ícones (icon/home.png) e vetores SVG (svg/world.svg), facilitando a organização e reutilização dos recursos.

Css: Uso do assets/css e src/css

O arquivo styles.css é o principal arquivo CSS do projeto, localizado na pasta assets/css. Ele contém estilos globais e pode ser modificado conforme necessário. Além disso, o arquivo index.css na pasta src/css é utilizado para personalizações específicas do projeto. É importante manter a organização e a separação de estilos para facilitar a manutenção e o desenvolvimento futuro.

assets/css/styles.css

O estilo global é aplicado através do arquivo styles.css, localizado na pasta assets/css. Ele contém definições de estilo que afetam toda a aplicação, garantindo consistência visual e facilitando a manutenção. Utilize este arquivo para definir cores, fontes, espaçamentos e outros elementos que serão compartilhados em várias partes do projeto.

assets/src/css/index.css

O arquivo index.css, localizado na pasta src/css, contém estilos pré-configurados e prontos para uso dinâmico. Ele foi projetado para facilitar a integração com scripts específicos do projeto, permitindo uma personalização eficiente e rápida. Além disso, o arquivo inclui suporte para CDNs prontas, oferecendo uma base sólida para o uso pessoal e garantindo compatibilidade com os principais recursos e bibliotecas utilizadas no projeto. Utilize este arquivo para estilizar elementos dinâmicos e integrar funcionalidades de forma prática e organizada.

MostrarDiv.js

A função mostrarDiv(id) tem como objetivo exibir uma div específica enquanto oculta todas as outras que possuem a classe .dm.

1. Selecionando todas as divs
O código usa document.querySelectorAll('.dm') para capturar todas as divs que possuem a classe .dm.

2. Ocultando todas as divs
Com um forEach, ele percorre cada div encontrada, adiciona a classe oculto para escondê-la e remove a classe fade-in para evitar animações antigas.

3. Exibindo a div desejada
A função obtém a div correspondente ao id passado como parâmetro e remove a classe oculto, tornando-a visível.

4. Adicionando animação
Após um pequeno atraso de 10 milissegundos, a classe fade-in é adicionada para criar um efeito suave de aparecimento.

Esse código permite alternar entre diferentes seções da página, garantindo que apenas uma div seja mostrada por vez, com uma transição suave.

PageLoad.js

Este código aguarda o carregamento do DOM para executar uma função que carrega dinamicamente conteúdos HTML em diferentes seções da página. A função recebe um nome de arquivo e um identificador de contêiner, monta o caminho correto do arquivo dentro de uma pasta específica e usa fetch() para buscá-lo. Se a resposta for bem-sucedida, o conteúdo é inserido no elemento correspondente; caso contrário, exibe um aviso ou erro no console. Em seguida, uma lista de elementos contendo arquivos e contêineres é percorrida, garantindo que cada seção da página seja preenchida automaticamente com seu respectivo conteúdo.
