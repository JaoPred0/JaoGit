Introdução: Crie e Publique Páginas no GitHub com Facilidade!

JaoGit é uma ferramenta prática para criar e publicar sites no GitHub Pages de forma rápida e simplificada. Com modelos prontos e suporte a HTML, CSS e JavaScript, ela facilita o desenvolvimento e aprimora a apresentação das suas páginas no GitHub.

Início rápido

Comece com o código padrão do HTML5 no arquivo index.html. No VS Code, digite ! ou html:5 e pressione Enter para gerar a estrutura básica automaticamente. Ou copia a estrutura abaixo.

Estrutura: Descubra o que está incluído no JaoGit.

As estruturas auxiliam na organização e na busca por arquivos, cada uma com seu propósito específico, como assets e base. Cada diretório contém elementos essenciais, e futuramente novas implementações facilitarão a criação de projetos e o upload no GitHub Pages.

Assets: Área de CSS, scripts e outros recursos globais

A pasta assets contém diretórios fixos, como css/styles.css e js/scripts.js. No entanto, é possível adicionar outros arquivos globais, como imagens (img/banner.png), ícones (icon/home.png) e vetores SVG (svg/world.svg), facilitando a organização e reutilização dos recursos.

Estrutura de Diretórios
  • assets
    • css
      • styles.css
    • js
      • scripts.js

Base: Estrutura Central da Página HTML

A pasta base contém os arquivos.html, responsáveis pela organização e estruturação do conteúdo da página.

Estrutura de Diretórios
  • base
    • arquivo.html

Src: Área de Configuração de Scripts e Estilos

A pasta src contém diretórios essenciais para os arquivos de estilo e scripts, como css/index.css e js/index.js. Embora seja possível adicionar ou remover arquivos conforme necessário, essa pasta é fundamental para a configuração e funcionamento da aplicação.

Estrutura de Diretórios
  • src
    • css
      • index.css
    • js
      • index.js
      • mostrarDiv.js
      • pageLoad.js



HTML: Estrutura do index.html
